Projects

Unity / C# / HLSL HoloDisplay ⌄

Collaborative immersive scientific visualisation system built as part of the CNRS CONTINUUM project (M2 I3D). Ports the JavaScript HoloCube prototype into a reusable Unity package, projecting real-time RGBD streams onto the three visible faces of a virtual cube.

  • Off-axis frustum projection that adapts dynamically to the observer's viewpoint
  • Custom HLSL shaders: depth reprojection in the fragment shader with SV_Depth write, and a separate depth-capture shader
  • Client-server architecture via Node.js WebSockets — DisplayClient sends camera matrices, RemoteRenderer streams back JPEG colour + raw RFloat depth textures
  • Matrix/texture snapshot system to keep shader input coherent across network latency
  • Additive Scene Loading separating local (HoloCube) and remote (scene to visualise) contexts
Unity C# HLSL Node.js WebSockets
OptiX / CUDA / C++ Path Tracing ⌄

GPU path tracer built with NVIDIA OptiX and CUDA as part of the M2 I3D programme. Implements physically-based light transport from scratch.

  • Hemisphere sampling using PSA formula with TBN orthonormal basis — concentrates samples near the surface normal for faster convergence and less noise
  • Full path tracing in the raygen shader — multi-bounce radiance accumulation, Russian roulette termination, environment map sampling on miss, anti-aliasing via per-pixel random offsets
  • Barycentric texture coordinate interpolation and interpolated normals; back-face detection with normal flipping
  • Direct lighting via explicit area light sampling — computes Li without relying on random hits, reducing noise significantly; occlusion tested with traceOcclusion
  • Lambertian BRDF with distance attenuation, light/surface angle cosine terms, and multi-sample averaging
OptiX CUDA C++ GLSL / HLSL

Research / OptiX Scientific Visualisation in Immersive Environments ⌄

TER research project (M2 I3D) exploring how immersive environments improve scientific visualisation over traditional 2D screens. Combined a literature review of 30+ years of research with a working prototype deployed in the CAVE system at iCube laboratory, Université de Strasbourg.

  • Studied key techniques: volume rendering, field visualisation, interactive feature extraction applied to fluid dynamics, medical imaging and astrophysical simulations
  • Integrated the iCube team's OptiX ray-casting volume renderer into the CAVE stereoscopic environment
  • Implemented stereoscopic rendering by computing per-eye camera offsets and CAVE screen coordinates, sending rays from each eye to each pixel of the physical screen panels
  • First approach: attempted to pass OptiX render textures into an OpenGL/easycppogl CAVE renderer — ran into compilation and GPU memory conflicts
  • Second approach: ported CAVE screen-coordinate and camera logic directly into the OptiX renderer — achieved a fixed-camera stereoscopic volume render on one screen at 60 fps per eye (120 Hz alternating lenses)
OptiX CUDA C++ OpenGL CAVE / VR
